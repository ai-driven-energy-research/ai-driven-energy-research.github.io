Overview

AIDER uses open peer review. Your name will be published alongside your review. This creates accountability and produces better, more constructive feedback.

Review Principles

  • Transparency: All reviews are public and signed
  • Constructiveness: Reviews should help authors improve their work, not just gatekeep
  • Reproducibility first: A paper with strong methodology but non-functional code is not acceptable. Reproducibility is a hard requirement.
  • AI use is welcome: You may use AI tools to assist your review. You must disclose this at the end of your review.

Two-Track Assessment

Track A: Research Quality

Evaluate the following and provide written assessment for each:

  • Novelty: Does this work make a new contribution? Is the problem well-motivated?
  • Methodology: Are the methods appropriate and correctly applied? Are assumptions clearly stated?
  • Results: Are claims supported by the evidence presented? Are comparisons fair? Are limitations acknowledged?
  • Clarity: Is the paper well-written and logically structured?
  • Relevance: Does this fall within AIDER's scope (AI + energy systems)?

Track B: Reproducibility

This is a hard gate. You must attempt to run the code.

  1. Clone the submission repository
  2. Follow the setup instructions in code/README.md
  3. Install dependencies from the provided dependency file
  4. Execute bash results/reproduce.sh
  5. Compare the generated outputs with the figures and tables in the paper

Report:

  • Did the code install without errors?
  • Did reproduce.sh execute successfully?
  • Do the generated results match the paper's claims?
  • Are there any discrepancies? If so, are they within acceptable tolerance?

Process Log Assessment

Check that the process log is substantive:

  • Is the research workflow clearly documented?
  • If AI tools were used, are session logs provided?
  • Are key human decisions recorded with reasoning?
  • Could another researcher understand the full process from the log alone?

Review Format

Post your review as a comment on the submission issue using this structure:

## Review — [Your Name]

### Summary
[2-3 sentence summary of the paper and your overall assessment]

### Track A: Research Quality
- Novelty: [assessment]
- Methodology: [assessment]
- Results: [assessment]
- Clarity: [assessment]
- Relevance: [assessment]

### Track B: Reproducibility
- Installation: [pass/fail, notes]
- Execution: [pass/fail, notes]
- Results match: [yes/no/partial, notes]

### Process Log
[Assessment of documentation completeness]

### Recommendation
[Accept / Minor revision / Major revision / Reject]

### AI Disclosure
[State any AI tools used during this review, or "No AI tools were used."]

Decision Criteria

  • Accept: Research is sound, code runs, results match, process is documented
  • Minor revision: Small issues in writing, code, or documentation that can be fixed quickly
  • Major revision: Significant methodological concerns or reproducibility issues that require substantial work
  • Reject: Fundamental flaws in methodology, code does not run after revision opportunity, or outside journal scope

Timeline

Please complete your review within 3 weeks of assignment. If you need more time, notify the editor. If you cannot complete the review, let us know as soon as possible so we can reassign.

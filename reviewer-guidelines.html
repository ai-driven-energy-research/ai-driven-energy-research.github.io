Overview

AIDER uses an AI-first review pipeline. Three automated agents handle reproducibility verification, pre-screening, and claim/reference auditing. A single editor reviews the AI reports, verifies flagged items, and makes the final editorial decision. Additional human reviewers may be invited at the editor's discretion.

Review Principles

  • Transparency: All reviews — automated and human — are public and signed
  • AI does the heavy lifting: Automated agents check reproducibility, flag bold claims, and audit references so the editor can focus on scientific judgement
  • Reproducibility first: A paper with strong methodology but non-functional code is not acceptable. Reproducibility is a hard requirement, verified automatically by CI
  • One editor is sufficient: The AI pipeline provides comprehensive reports; the editor makes the final call. Additional reviewers may be invited but are not required

Automated Review Pipeline

Three AI agents run automatically when a submission issue is opened:

Stage 1: Reproducibility Check

The CI pipeline automatically:

  • Verifies the repository structure (paper/, code/, data/, process-log/, etc.)
  • Checks for dependency management files
  • Installs dependencies and executes reproduce.sh
  • Reports pass/fail for each check

Stage 2: AI Pre-Screening

An AI agent reads the submission and generates a report covering:

  • Paper summary and key claims
  • Completeness check (manuscript, code, data, process log)
  • Code observations (hardcoded paths, missing seeds, suspicious patterns)
  • Process log assessment (substantive vs. boilerplate)
  • Attention points for the editor

Stage 3: AI Deep Review

A second AI agent performs a claim and reference audit:

  • Claim verification: Extracts bold claims ("first", "novel", "unprecedented", "outperforms") and rates each as HIGH / MEDIUM / LOW severity
  • Reference accessibility: Resolves DOIs and flags paywalled references the editor must download to verify
  • Missing references: Identifies .bib entries without DOIs or URLs

Editor's Role

After the three automated stages complete, the editor:

  1. Reads the AI-generated reports (pre-screening + claim audit + reference audit)
  2. Focuses on HIGH-severity claims flagged by the deep review
  3. Downloads and verifies any paywalled references the AI could not access
  4. Assesses scientific merit, novelty, and methodology
  5. Optionally invites additional human reviewers for specialised expertise
  6. Posts the editorial decision on the submission issue

Review Format

Post your review as a comment on the submission issue using this structure:

## Editorial Review — [Your Name]

### Summary
[2-3 sentence summary of the paper and your overall assessment]

### Research Quality
- Novelty: [assessment]
- Methodology: [assessment]
- Results: [assessment]
- Clarity: [assessment]
- Relevance: [assessment]

### Claim Verification
[Response to AI-flagged claims — verified / unverified / concern]

### Paywalled References
[For each flagged reference — verified content matches citation / discrepancy found]

### Process Log
[Assessment of documentation completeness]

### Recommendation
[Accept / Minor revision / Major revision / Reject]

### AI Disclosure
[State any AI tools used during this review, or "No AI tools were used."]

Decision Criteria

  • Accept: Research is sound, code runs, results match, process is documented, claims are supported
  • Minor revision: Small issues in writing, code, or documentation that can be fixed quickly
  • Major revision: Significant methodological concerns, unverified claims, or reproducibility issues that require substantial work
  • Reject: Fundamental flaws in methodology, code does not run after revision opportunity, unsupported bold claims, or outside journal scope

Timeline

The AI pipeline completes within minutes of submission. The editor should aim to post their decision within 2 weeks. If additional reviewers are invited, they have 3 weeks to respond.

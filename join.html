Hey — I'm starting an academic journal called AI-Driven Energy Research (AIDER), and I want you to be part of the founding editorial board.

Before you close this tab — this is not a traditional journal. There's no publisher behind it, no fees, no bureaucracy. It's a bunch of PhD students building something new on GitHub. The whole thing costs $0 to run.

The Problem

You've read papers where the results can't be reproduced. You've tried to find the code — it doesn't exist. You've emailed the authors — no reply. The paper is cited 200 times but nobody can actually use it.

Meanwhile, we're all using AI to help with our research — writing code, running experiments, even drafting papers. But we're hiding it, because journals haven't caught up.

What AIDER Does Differently

Every paper must be fully reproducible. Code, data, and a script that regenerates every figure. If the code doesn't run, the paper is rejected. No exceptions.
Every paper must show how it was made. If you used AI agents to help write or code — share the logs. If you made a key decision — document it. The research process is part of the contribution.
AI is encouraged, not hidden. We think AI-driven research is the future. AIDER is built for researchers who actually work this way.
Completely free. No money involved. Ever. No APCs, no subscriptions, no sponsorships. Everything runs on GitHub (free), Zenodo (free), and volunteer time.

Why Join Now

This is the founding team. We're not adding big names to decorate a masthead — we want people who will actually do things. You'd be shaping the journal from day one: the review process, the standards, what gets published, how it works.

  • You get to put "Founding Editorial Board Member" on your CV — that's a real line item for academic jobs and postdoc applications
  • You build something that might genuinely change how research is published in our field
  • You work with friends, not a faceless editorial system
  • It takes very little time (see below)

What You'd Actually Do

Editorial Board Member

~2-3 hours per month
  • Review 1-2 papers per quarter (when submissions come in)
  • For each review: read the paper, clone the repo, try to run the code, write your assessment
  • Give input on journal policies and direction (group chat / occasional call)
  • Optionally submit your own work as a founding paper

That's it. No admin, no typesetting, no chasing authors. GitHub handles the workflow.

How It Works (30-Second Version)

  1. Author creates a GitHub repo from our template (paper + code + data + process log)
  2. Author opens an Issue to submit
  3. We assign two reviewers — reviews happen as public GitHub comments
  4. If the paper is scientifically sound AND the code runs → accepted
  5. Paper repo gets forked into the AIDER org and listed on the website

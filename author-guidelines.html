Overview

AIDER accepts original research at the intersection of artificial intelligence and energy systems. Every submission must be a complete, executable research package.

Manuscript Preparation

Format

Manuscripts must be submitted as source files (LaTeX or Markdown), not compiled PDFs. Place all manuscript files in the paper/ directory of your repository. There is no strict page limit, but clarity and conciseness are expected.

Structure

Papers should follow standard academic structure: Abstract, Introduction, Methodology, Results, Discussion, Conclusion, References. Variations are acceptable where the research warrants it.

Language

All submissions must be in English. AI-assisted writing is permitted and must be disclosed in the process log.

Repository Requirements

Start by creating a repository from the AIDER paper template. Your repository must contain:

Code (code/)

  • All source code required to reproduce every result in the paper
  • A README.md with clear setup and execution instructions
  • Dependency management via requirements.txt, environment.yml, pyproject.toml, or Dockerfile
  • Code must execute without errors on a clean environment

Data (data/)

  • All data necessary to reproduce results
  • If data exceeds GitHub's file size limits, host on Zenodo or HuggingFace Datasets with a persistent DOI
  • A README.md describing data format, source, collection method, and license
  • Synthetic or generated data must include the generation script

Process Log (process-log/)

This is what makes AIDER unique. You must document how the paper was produced:

  • AI sessions (process-log/ai-sessions/): Full transcripts or logs of any AI agent sessions used during research — including prompts, outputs, and tool use. Accepted formats: text logs, JSON exports, JSONL conversation dumps, or screenshots with descriptions
  • Human decisions (process-log/human-decisions/): A timestamped log of significant human decisions — parameter choices, methodology pivots, manual corrections, data cleaning decisions. Can be a simple Markdown file with dated entries
  • Overview (process-log/README.md): A narrative summary of the research workflow — which stages used AI, which were manual, and how human and AI contributions interacted

Reproducibility Script (results/reproduce.sh)

  • A single script that regenerates all key figures and tables from the paper
  • Must exit with code 0 on success
  • Random seeds must be fixed for deterministic output
  • Execution time should be documented in the script header (the CI allows up to 60 minutes)

Reproducibility Checklist (REPRODUCIBILITY.md)

Complete every item in the provided checklist before submitting. Incomplete checklists will result in desk rejection.

AI Disclosure

AIDER encourages the use of AI tools throughout the research process. All AI use must be documented:

  • Name and version of each AI tool used (e.g., Claude 3.5 Sonnet, GPT-4, GitHub Copilot)
  • How each tool was used (writing, coding, analysis, literature review, etc.)
  • Session logs in process-log/ai-sessions/

Failure to disclose AI use when evidence of it is found during review will result in rejection.

Licensing

  • Paper: CC-BY 4.0 — authors retain copyright, anyone may reuse with attribution
  • Code: MIT or Apache 2.0 (author's choice)
  • Data: CC0 (public domain) or CC-BY 4.0

How to Submit

  1. Create your repository from the paper template
  2. Populate all required directories and complete the reproducibility checklist
  3. Open a submission issue
  4. Fill in the submission form with your repository link and details
  5. The automated reproducibility pipeline will run within minutes
  6. An editor will assign reviewers once the automated check passes

After Submission

  • Automated check: The CI pipeline verifies repository structure, installs dependencies, and runs reproduce.sh. If this fails, you will be asked to fix issues before human review begins.
  • Review: Two reviewers (AI-assisted + human editor) will assess methodology and reproducibility. Reviews are posted publicly on your repository.
  • Revision: If revisions are requested, update your repository and comment on the submission issue. There is no limit on revision rounds.
  • Acceptance: Your repository is forked into the AIDER GitHub organisation and assigned a Zenodo DOI.
